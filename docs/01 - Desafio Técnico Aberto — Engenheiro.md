Estamos com uma oportunidade aberta e, ao invés de um processo seletivo tradicional, resolvi propor algo mais mão na massa: um desafio técnico real, com regras de negócio de verdade, integração com dados da B3, Kafka, MySQL e .NET Core.

O desafio? Construir um Sistema de Compra Programada de Ações — sim, aquele produto onde o cliente define um valor mensal e o sistema automaticamente compra uma carteira recomendada de 5 ações, distribui proporcionalmente, calcula IR, faz rebalanceamento... o pacote completo.

Para facilitar a vida de quem topar o desafio, preparei um kit bem completo de materiais de apoio:

📄 Documento do Desafio Técnico — especificação completa com todas as funcionalidades, requisitos técnicos e critérios de avaliação
📖 Glossário de Termos — do básico ao avançado, para ninguém ficar perdido com os termos do mercado financeiro
📊 Diagrama de Negócios — visão geral dos fluxos do sistema (draw.io)
🗃️ Diagrama de Entidade-Relacionamento (ER) — modelo de dados sugerido com 10 tabelas e todos os relacionamentos (draw.io)
🔄 Diagrama de Sequência — fluxo passo a passo do Motor de Compra (18 etapas) e do Rebalanceamento (12 etapas) (draw.io)
📑 Layout do Arquivo COTAHIST da B3 — explicação detalhada do formato do arquivo de cotações + exemplo de parse em C#
📋 Regras de Negócio Detalhadas — 70 regras numeradas com exemplos numéricos completos, casos de borda e cenários de IR
🔌 Exemplos de Contratos de API — request/response JSON de todos os endpoints principais
